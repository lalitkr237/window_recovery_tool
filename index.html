Overview

How It Works

At the Windows login screen, there is an Accessibility (Utilman) button in the bottom-right corner. Before Windows boots, we temporarily replace utilman.exe with cmd.exe — so clicking that button opens a Command Prompt before login, giving admin-level access to reset any local password.

Files involved: C:\Windows\System32\utilman.exe and C:\Windows\System32\cmd.exe
Step 01

1 Boot Into Recovery CMD

Force your PC into the Windows Recovery Environment by holding Shift while clicking Restart, or by interrupting boot 2–3 times (force shutdown during startup).

Navigate to: Troubleshoot → Advanced Options → Command Prompt

Command Prompt — Recovery Environment
X:\Sources> C: C:\> cd Windows\System32 C:\Windows\System32> // You are now in System32
Step 02

2 Swap utilman.exe with cmd.exe

Backup the original utilman.exe by renaming it, then rename cmd.exe to take its place:

Command Prompt — System32
// Backup original utilman C:\Windows\System32> ren utilman.exe utilman1.exe   // Put cmd.exe in its place C:\Windows\System32> ren cmd.exe utilman.exe   C:\Windows\System32> exit // PC will restart normally
What happened: utilman.exe is backed up as utilman1.exe. The Accessibility button now launches CMD instead of the accessibility menu.
Step 03

3 Reset the Password

After restart, at the login screen, click the Accessibility icon (bottom-right corner). A CMD window will open with admin-level access.

Command Prompt — Login Screen (Pre-Auth)
// Option A — GUI (easier) C:\Windows\System32> control userpasswords2 // Select your user → Reset Password → enter new password   // Option B — command line C:\Windows\System32> net user [username] [newpassword] The command completed successfully.   // To list all local accounts: C:\Windows\System32> net user User accounts for \\DESKTOP-XXXXX ----------------------------------------------- Administrator John Guest

Close CMD and log in with your new password. ✅

Step 04 — CRITICAL

4 Restore System Files

After logging in, open CMD as Administrator (Win + R → cmd) and reverse the swap. Leaving the swap in place is a security vulnerability — anyone can open CMD from your login screen.

Command Prompt — Run as Administrator
C:\> cd Windows\System32   // Delete the fake utilman (which is actually cmd) C:\Windows\System32> del utilman.exe   // Restore original utilman from backup C:\Windows\System32> ren utilman1.exe utilman.exe   // Verify — should show only one utilman.exe C:\Windows\System32> dir utilman*.exe 03/15/2026 11:02 PM 315,392 utilman.exe 1 File(s) 315,392 bytes   ✓ Restore complete. System is secure.
✅ Done! Both utilman.exe and cmd.exe are back to their original state. Restart your PC to confirm everything works.
Troubleshooting

Common Errors & Fixes

Error: "A duplicate file name exists, or the file cannot be found" when running ren utilman.exe cmd.exe

This means cmd.exe already exists in System32. Fix:

Fix — cmd.exe already exists
// cmd.exe already exists — just clean up utilman files C:\Windows\System32> del utilman.exe C:\Windows\System32> ren utilman1.exe utilman.exe Done. Both files restored.
